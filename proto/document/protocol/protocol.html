StoreIt Protocol v0.1

This protocol is ASCII. Every command ends with \r\n.

1. JSON object description

1. 1. File object:



1. Session:

1. 1. JOIN

[client]->[server] This is the first request to make whenever a client wants to get online.

command: JOIN username port file_json_object

1. 2. LEAVE


[client]->[server] A client should send this to the master server to leave the network

command: LEAVE

2. Data:


2. 1. CHDELETE


[server -> client] Delete a chunk from the hidden store

command: CHDELETE chunk_hash

2. 2. CHSEND


[server -> client] Tells a client to wait for a specific client to send it a chunk (only in the case of chunks that don't belongs to a user), or to send a chunk to another client

command: CHSEND send chunk_hash ip_address:port

send: 0: should wait for incoming chunk 1: should send the chunk to the specified client
chunk_hash: the chunk's hash
ip_address:port: the ip address and port of the client that shoud be contacted

2. 3. CHSTORE


[client -> client] Send a chunk to a client
command: CHSTORE data_length row_data

2. 4. FDELETE

[client or server] Delete a file
command: FDELETE file_path

2. 5. FADD

[client or server] Add a file to the user tree. If it is sent from the server, the user should create the file and wait further instructions to fetch the file data. Otherwise if it is sent from the client, the server will attempt to add the file to the user virtual tree. It will then send further instructions to dispatch the data on the network.
command: FADD json_file_object

2. 6. FUPDATE

[client or server] update a file
command: FUPDATE json_file_object
